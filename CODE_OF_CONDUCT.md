# Code of Conduct

## Our Commitment

We are committed to fostering an open, welcoming, inclusive, and respectful community for everyone. Memory Palace exists to help developers of all experience levels collaborate on and learn from a real, working project.

We expect all community members to contribute to a positive environment where ideas can be shared openly and respectfully.

---

## Our Standards

Examples of behavior that contribute to a positive community include:

* Being respectful and considerate toward others.
* Providing constructive feedback.
* Welcoming newcomers and helping them learn.
* Respecting differing opinions and experiences.
* Focusing discussions on improving the project.
* Accepting feedback with professionalism.
* Using inclusive and respectful language.
* Collaborating in good faith.

---

## Unacceptable Behavior

The following behaviors are not tolerated:

* Harassment, discrimination, or hate speech.
* Personal attacks, insults, or abusive language.
* Trolling, intimidation, or deliberate disruption.
* Publishing private or confidential information without permission.
* Spam, malicious content, or misleading contributions.
* Plagiarism or intentional misrepresentation of work.
* Any behavior that creates an unsafe or hostile environment.

---

## Scope

This Code of Conduct applies to all project spaces, including:

* GitHub Issues
* Pull Requests
* Discussions
* Project documentation
* Any other official communication channels related to Memory Palace

---

## Contribution Expectations

When contributing to Memory Palace, contributors should:

* Follow the project's contribution guidelines in `CONTRIBUTING.md`.
* Write clean and maintainable code.
* Keep pull requests focused on a single change.
* Document new functionality when appropriate.
* Review feedback respectfully.
* Help maintain a welcoming community.

---

## Enforcement

The project maintainer is responsible for interpreting and enforcing this Code of Conduct.

The maintainer may take appropriate action in response to unacceptable behavior, including:

* Issuing warnings.
* Requesting changes to behavior.
* Removing comments, issues, or pull requests.
* Temporarily restricting participation.
* Permanently banning individuals from the project if necessary.

Enforcement decisions are made at the maintainer's discretion to protect the health of the community.

---

## Reporting Issues

If you experience or witness unacceptable behavior, please report it privately by contacting the maintainer directly through GitHub (@aayushirajesh).

Reports will be handled respectfully, confidentially where possible, and investigated promptly.
